# 素材归档（Raw）

> 原始素材的只读归档区：PRD 原件、会议纪要、需求截图、接口契约原文等。

---

## 规则

1. **只读**：本目录内容是「原件」，**不在此处编辑加工**。需要提炼的结论写到 `system/`、`modules/`、`insights/` 等处。
2. **保真**：尽量保留原始格式与出处（来源、日期、提供人）。
3. **可追溯**：每份素材建议在文件名或开头标注日期与来源，如 `2026-06-25_PRD_反馈模块.md`。
4. **归档即留痕**：新增素材后在 `../log.md` 追加一行。

---

## 建议的素材分类

| 子类 | 放什么 |
|------|--------|
| PRD / 需求 | 产品需求文档原件、需求变更说明 |
| 会议纪要 | 评审、对齐、复盘的纪要 |
| 接口契约 | 后端给的接口约定原文（项目根目录已有 `API接口契约.md`，可在此建链接或副本） |
| 设计稿 | UI 稿、风格稿说明（根目录有 `风格稿.html`） |
