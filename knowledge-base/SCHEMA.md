# SCHEMA —— 知识库文档规范（机器可校验）

> 本文件定义知识库中所有文档的格式标准。目标：让人和机器（LLM、校验脚本）都能稳定解析。

---

## 1. 通用规则

- 文件编码：UTF-8，无 BOM。
- 换行：LF。
- 语言：默认中文（术语、代码标识符保留原文）。
- 文件名：小写英文 + 连字符（kebab-case），如 `call-chains.md`。
- 每个文档必须以 **YAML Front Matter** 元数据开头（见下）。

---

## 2. Front Matter（元数据头）

每个 `.md` 文档（`README.md`、`SCHEMA.md`、`MAINTENANCE.md`、`log.md` 除外）必须以如下元数据开头：

```yaml
---
title: 文档标题
category: system | modules | pitfalls | conventions | insights | raw
status: draft | stable | deprecated
owner: 负责人（姓名或工号）
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [标签1, 标签2]
---
```

字段说明：

| 字段 | 必填 | 取值 | 说明 |
|------|------|------|------|
| `title` | 是 | 字符串 | 文档标题，单行 |
| `category` | 是 | 枚举 | 必须与所在目录一致 |
| `status` | 是 | `draft`/`stable`/`deprecated` | 文档成熟度 |
| `owner` | 是 | 字符串 | 谁对内容负责 |
| `created` | 是 | 日期 | 创建日期 |
| `updated` | 是 | 日期 | 最后更新日期 |
| `tags` | 否 | 字符串数组 | 便于检索 |

---

## 3. 正文结构

- 一级标题 `#` 全文只出现一次，且应等于 `title`。
- 章节用 `##`、`###` 逐级下钻，**不跳级**。
- 代码块必须标注语言：` ```js `、` ```bash ` 等。
- 引用源码位置用相对路径 + 行号：`src/api/request.js:32`。
- 表格列对齐统一用 `|`，表头与分隔线必备。

---

## 4. 坑点（pitfalls）专用规范

坑点文档每条记录必须包含以下结构化字段：

```markdown
### [P0|P1|P2] 简短标题

- **现象**：发生了什么（可观察的症状）
- **根因**：为什么会发生
- **影响范围**：涉及哪些模块/页面
- **规避/解决**：怎么绕过或修复
- **关联代码**：`src/xxx.vue:行号`
- **状态**：未解决 | 已规避 | 已修复
```

分级标准：

| 级别 | 含义 | 例子 |
|------|------|------|
| **P0** | 阻断性，会导致功能不可用或数据错误 | 接口拆包逻辑错误导致页面白屏 |
| **P1** | 严重，影响体验但有可接受的临时方案 | 分页参数命名不一致需手动适配 |
| **P2** | 轻微，优化项 | 文案不统一、样式细节 |
