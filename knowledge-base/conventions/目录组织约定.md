# 目录组织约定

> 项目目录与文件的组织原则，确保结构清晰、易维护。

---

## 1. 核心原则

### 1.1 按功能领域分层，不按技术类型

**✅ 正确**：按业务领域组织

```
api/
  ├── agent.js          # 智能体管理接口
  ├── category.js       # 分类管理接口
  └── feedback.js       # 用户反馈接口

views/
  ├── AgentManagement.vue
  ├── CategoryManagement.vue
  └── FeedbackRecords.vue
```

**❌ 错误**：按技术类型组织

```
components/
  ├── tables/           # 所有表格组件
  ├── forms/            # 所有表单组件
  └── dialogs/          # 所有弹窗组件
```

**理由**：
- 按领域组织更易定位：修改「智能体管理」功能，只需找 `api/agent.js` + `views/AgentManagement.vue`
- 按技术类型组织需要跨多个目录才能完成一个功能

### 1.2 一个业务 = 一组文件

**标准模式**：

```
智能体管理功能 =
  ├── src/api/agent.js              # 接口层
  ├── src/views/AgentManagement.vue  # 页面层
  └── src/router.js                  # 路由配置（一条路由记录）
```

**约定**：
- API 文件与 View 文件**命名对应**（agent ↔ AgentManagement）
- 路由 `name` 与 View 文件名一致（`name: 'Agents'` ↔ `AgentManagement.vue`）

## 2. 文件命名规范

### 2.1 组件文件（.vue）

**规则**：PascalCase（大驼峰）

| 类型 | 命名 | 示例 |
|------|------|------|
| 页面组件 | 领域名 + Management/Config/Records | `AgentManagement.vue`<br>`QuickQuestionConfig.vue`<br>`FeedbackRecords.vue` |
| 布局组件 | 简洁名词 | `Layout.vue` |
| 公共组件 | 描述性名词 | `UserAvatar.vue`<br>`StatusTag.vue` |

**后缀约定**：
- `Management`：管理类页面（增删改查）
- `Config`：配置类页面
- `Records`：记录查看页面
- `Warnings`：预警/告警页面

### 2.2 脚本文件（.js）

**规则**：camelCase（小驼峰）

| 类型 | 命名 | 示例 |
|------|------|------|
| API 模块 | 领域名 | `agent.js`<br>`category.js`<br>`quickQuestion.js` |
| 工具函数 | 功能描述 | `request.js`<br>`format.js`<br>`validate.js` |
| 配置文件 | 配置内容 | `router.js`<br>`main.js` |

### 2.3 样式文件（.css）

**规则**：kebab-case（连字符）或 index.css

```
styles/
  ├── index.css         # 全局样式入口
  ├── variables.css     # CSS 变量（如有独立）
  └── utilities.css     # 工具类（如有独立）
```

## 3. 目录层级约定

### 3.1 扁平优于嵌套

**✅ 推荐**：最多 2-3 层

```
src/
  ├── api/              # 1 层
  │   ├── agent.js
  │   └── category.js
  ├── views/            # 1 层
  │   ├── AgentManagement.vue
  │   └── CategoryManagement.vue
  └── layout/           # 1 层
      └── Layout.vue
```

**❌ 避免**：过度嵌套

```
src/
  ├── modules/
  │   ├── agent/
  │   │   ├── api/
  │   │   │   └── index.js
  │   │   └── views/
  │   │       └── Management.vue
  │   └── category/
  │       └── ...
```

**理由**：扁平结构更易浏览，IDE 文件树更清晰。

### 3.2 公共基础设施独立

**公共模块**（被多个业务复用）：

```
src/
  ├── api/request.js    # 所有接口共用
  ├── layout/Layout.vue # 所有页面共用
  ├── styles/index.css  # 全局样式
  └── router.js         # 全局路由
```

**业务模块**（特定功能专用）：

```
src/
  ├── api/agent.js
  ├── views/AgentManagement.vue
```

## 4. 各目录职责

### 4.1 src/ 结构

```
src/
├── api/              # 接口层：封装所有后端调用
├── layout/           # 布局组件：页面外壳
├── views/            # 页面组件：业务页面
├── styles/           # 全局样式：设计系统
├── main.js           # 应用入口
├── App.vue           # 根组件
└── router.js         # 路由配置
```

**禁止出现的目录**（当前项目不使用）：
- `store/`：无全局状态管理（Vuex/Pinia）
- `utils/`：工具函数按需添加，当前无需
- `composables/`：组合式函数按需添加，当前无需
- `components/`：公共组件按需添加，当前无需

### 4.2 根目录结构

```
后台管理/
├── src/              # 源代码
├── mock/             # Mock 数据
├── knowledge-base/   # 知识库（团队共享知识）
├── workspace/        # 工作台（临时任务）
├── dist/             # 构建产物（生成，不入版本控制）
├── node_modules/     # 依赖包（生成，不入版本控制）
├── index.html        # HTML 入口
├── vite.config.js    # 构建配置
├── package.json      # 依赖清单
├── .env.*            # 环境变量
└── API接口契约.md     # 素材文档
```

## 5. 新增模块 SOP

### 5.1 新增业务功能

**标准流程**：

1. **确认领域名**：如「通知管理」→ `notification`
2. **创建 API 文件**：`src/api/notification.js`
3. **创建页面组件**：`src/views/NotificationManagement.vue`
4. **注册路由**：在 `router.js` 添加路由记录
5. **添加菜单**：在 `layout/Layout.vue` 添加侧边栏导航项
6. **更新知识库**：
   - 在 `knowledge-base/modules/inventory.md` 登记新模块
   - 在 `knowledge-base/system/URL路由.md` 添加路由记录
   - 在 `knowledge-base/log.md` 追加变更记录

### 5.2 新增公共组件

**适用场景**：同一组件被 ≥ 3 个页面复用时提取。

**创建位置**：
```
src/
  └── components/
      └── StatusTag.vue    # 公共状态标签组件
```

**命名规范**：
- 文件名：PascalCase
- 组件名：与文件名一致
- 前缀：可选（如 `Base-`、`App-`）

## 6. 文件拆分原则

### 6.1 何时拆分

| 场景 | 操作 |
|------|------|
| 单文件超过 500 行 | 考虑拆分逻辑或模板 |
| 同一逻辑重复 ≥ 3 次 | 提取为 composable 或 util |
| 同一组件被 ≥ 3 个页面使用 | 提取为公共组件 |

### 6.2 拆分方式

**模板过长**：拆分为子组件

```
AgentManagement.vue（主页面）
  ├── AgentTable.vue      # 表格部分
  ├── AgentForm.vue       # 表单部分
  └── AgentFilters.vue    # 筛选栏部分
```

**逻辑重复**：提取为 composable

```
src/
  └── composables/
      ├── usePagination.js  # 分页逻辑复用
      └── useDialog.js      # 弹窗逻辑复用
```
