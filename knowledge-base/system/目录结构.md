# 目录结构说明

> 项目完整的目录组织方式与各目录职责。

---

## 1. 整体结构

```
后台管理/
├── index.html                   # HTML 入口
├── vite.config.js               # Vite 构建配置（Mock 开关、代理配置）
├── package.json                 # 依赖清单与脚本
├── package-lock.json            # 依赖锁定版本
├── .env.development             # 开发环境变量
├── .env.production              # 生产环境变量
├── API接口契约.md                # 后端接口文档
├── 风格稿.html                   # UI 设计规范
│
├── src/                         # 源代码目录（核心）
│   ├── main.js                  # 应用入口
│   ├── App.vue                  # 根组件
│   ├── router.js                # 路由配置
│   ├── api/                     # 接口层
│   ├── layout/                  # 布局组件
│   ├── views/                   # 页面组件
│   └── styles/                  # 全局样式
│
├── mock/                        # Mock 数据（开发演示用）
├── dist/                        # 构建产物（生成，不入版本控制）
├── node_modules/                # 依赖包（生成，不入版本控制）
├── knowledge-base/              # 知识库（团队长期知识）
└── workspace/                   # 工作台（临时任务区）
```

---

## 2. src/ 详细结构

### 2.1 入口文件

| 文件 | 职责 | 关键代码 |
|------|------|----------|
| `main.js` | 应用启动入口，注册全局插件与组件 | 注册 Element Plus、全部图标、路由；挂载到 `#app` |
| `App.vue` | 根组件，仅包含 `<router-view />` | 简洁的视图容器 |
| `router.js` | 路由表定义 | 5 个业务页面 + Layout 布局壳 |

### 2.2 api/ —— 接口层

**职责**：封装所有后端接口调用，业务页面不直接用 axios。

```
api/
├── request.js          # axios 统一实例（拦截器、拆包、错误处理）
├── agent.js            # 智能体管理接口
├── category.js         # 分类管理接口
├── quickQuestion.js    # 快捷问题配置接口
├── feedback.js         # 用户反馈接口
└── warning.js          # 心理风险预警接口
```

**约定**：
- 每个业务领域一个文件（与 `views/` 一一对应）
- 函数命名：`getXxx`、`createXxx`、`updateXxx`、`deleteXxx`
- 每个导出函数都有 JSDoc 注释
- 所有请求复用 `request.js` 实例

### 2.3 layout/ —— 布局组件

```
layout/
└── Layout.vue          # 后台管理主布局（侧边栏 + 顶栏 + 内容区）
```

**Layout.vue 结构**：
- **侧边栏**（`.sidebar`）：品牌区、导航菜单、用户信息
- **主容器**（`.main-container`）：顶栏面包屑 + 页面内容区 `<router-view />`

### 2.4 views/ —— 页面组件

**职责**：业务页面，与路由一一对应。

```
views/
├── AgentManagement.vue           # 智能体管理
├── CategoryManagement.vue        # 分类管理
├── QuickQuestionConfig.vue       # 快捷问题配置
├── FeedbackRecords.vue           # 用户反馈
└── PsychologicalWarnings.vue     # 心理风险预警
```

**通用结构模式**：
1. 搜索筛选栏（`.card-toolbar`）
2. 数据表格（`el-table`）
3. 分页器（`el-pagination`）
4. 弹窗（`el-dialog`，新增/编辑表单）

### 2.5 styles/ —— 全局样式

```
styles/
└── index.css          # 全局设计系统（Design Tokens、组件覆盖、工具类）
```

**关键内容**：
- CSS 变量（`:root`）：品牌色、中性色、圆角、阴影
- Element Plus 主题覆盖
- 工具类（`.card-compact`、`.flex-between` 等）

## 4. 配置文件

| 文件 | 职责 |
|------|------|
| `vite.config.js` | 构建配置：Mock 开关、后端代理、端口、自动打开浏览器 |
| `package.json` | 依赖清单、脚本（dev/build/preview） |
| `.env.development` | 开发环境变量（`VITE_API_BASE_URL`、`VITE_USE_MOCK`） |
| `.env.production` | 生产环境变量（关闭 Mock，指向真实后端） |

## 5. 文档与素材

| 文件/目录 | 性质 | 职责 |
|-----------|------|------|
| `API接口契约.md` | 素材 | 后端接口约定（响应格式、字段说明） |
| `风格稿.html` | 素材 | UI 设计规范（颜色、字体、组件样式） |
| `knowledge-base/` | 持久知识 | 团队共享的稳定知识（架构、约定、经验） |
| `workspace/` | 临时任务 | 进行中的工作草稿，用完可清理 |

## 6. 目录组织原则

1. **按功能领域分层，不按技术类型**：
   - ✅ `api/agent.js` + `views/AgentManagement.vue`（同一领域，易定位）
   - ❌ 不采用 `components/tables/`、`components/forms/` 这种技术维度分类

2. **一个业务 = 一个 view + 一个 api 文件**：
   - 新增功能时同步新增两个文件，命名对应

3. **公共基础设施独立**：
   - `api/request.js`：所有接口共用
   - `layout/Layout.vue`：所有页面共用
   - `styles/index.css`：全局样式

4. **配置集中**：
   - 环境变量放 `.env.*`
   - 构建配置放 `vite.config.js`
   - 不在源码中硬编码 URL 或开关
